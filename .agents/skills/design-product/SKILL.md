# OBJECTIVE

Create or update the Product Requirements Document.

# INPUTS

docs/Vision.md

docs/Constraints.md

docs/Research_Report.md

Optional:

docs/PRD.md

# OUTPUT

docs/PRD.md

# UPDATE STRATEGY

If PRD.md exists:

Treat it as approved product definition.

Preserve unchanged requirements.

Modify only impacted sections.

Avoid document-wide rewrites.

# PRD MUST INCLUDE

Problem Statement

Target Users

User Stories

Functional Requirements

Non-functional Requirements

MVP Scope

Future Scope

Success Metrics

# RULES

Follow Vision.md.

Follow Research_Report.md.

Prefer smallest viable scope.

Avoid feature creep.

Prefer incremental updates.

All outputs must be written in Markdown format.
