# OBJECTIVE

Translate product requirements into a structured, implementation-ready UI/UX design artefact set.

# INPUTS

docs/Vision.md

docs/Constraints.md

docs/Research_Report.md

docs/PRD.md

Optional:

docs/User-Flows.md

docs/Screen-Specs.md

docs/Visual-Guidelines.md

docs/UI-Layouts.md

# OUTPUTS

docs/User-Flows.md

docs/Screen-Specs.md

docs/Visual-Guidelines.md

docs/UI-Layouts.md

# UPDATE STRATEGY

If optional documents are provided:

Treat them as approved project history.

Preserve unchanged decisions.

Modify only affected sections.

Avoid unnecessary rewrites.

# User-Flows.md MUST INCLUDE

User goals & entry points

Navigation paths & decision points

Success outcomes

Alternate & error flows

# Screen-Specs.md MUST INCLUDE

Screen name, purpose, & user goals

Components & user interactions

Validation rules & data requirements

Loading, empty, error, & success states

Navigation actions

# Visual-Guidelines.md MUST INCLUDE

Brand personality & design principles

Colour palette & typography rules

Spacing guidelines & layout principles

Component styling rules

Accessibility & responsive design guidelines

# UI-Layouts.md MUST INCLUDE

Page structure & component placement

Responsive layouts (Desktop, Tablet, Mobile)

Navigation patterns & layout hierarchy

Simple diagrams, wireframes, tables, or structured descriptions

# RULES

Do not design system architecture or APIs.

Do not design databases or module_map.

Do not define deployment strategies or implementation details.

Prioritise simplicity, clarity, accessibility, consistency, and mobile-first thinking.

Focus on creating specifications optimized for Frontend Agent implementation without missing requirements.

All outputs must be written in Markdown format.
